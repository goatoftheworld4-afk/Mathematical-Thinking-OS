# Common Viewpoints OS

## 1. Purpose

共通着眼点OSは、数学の分野に依存せず、多くの入試問題に横断的に現れる構造を整理したものである。

問題を見たとき、最初から公式や典型解法を探すのではなく、まず問題文に含まれる条件を数学的な構造へ翻訳する。

基本的な流れは次である。

```text
問題文の条件
↓
条件の数学的解釈
↓
発動する共通着眼点
↓
候補となる操作
↓
分野固有の知識

共通着眼点は、解法そのものではない。

共通着眼点の役割は、既習の公式・定理・解法を呼び出すための索引を与えることである。

2. Entry Format

各着眼点は、原則として次の項目を持つ。

・ID
・名称
・定義
・発動条件
・条件の解釈
・自然に候補となる操作
・候補の優先順位
・他の着眼点との関係
・非発動条件
・よくある誤用
・検証時の問い
・復習カード

候補の優先順位は絶対的なものではない。

問題文の構造、求める対象、計算量、定義域、場合分けの複雑さによって変化する。

OS-001 対称性
定義

複数の対象を入れ替え、反転し、回転し、または一定の変換を施しても、問題の条件や求める量が本質的に変化しない構造を対称性という。

対称性は、対象を区別する必要がないこと、または自由度を削減できることを示唆する。

発動条件

次のような条件があるときに候補となる。

・変数 a,b,c などが同じ形で現れる
・変数を入れ替えても条件式が変わらない
・和、積、基本対称式のみで条件が与えられている
・図形が線対称、点対称、回転対称である
・複数の点、辺、領域が同じ役割を持つ
・確率や場合の数で各対象が同等に扱われる
・数列や漸化式で複数の状態が対等に現れる
・条件の解釈

対称性がある場合、対象を個別に追跡するよりも、共通する量や対称性を保つ量を用いた方が自然である。

また、必要に応じて代表的な一つの場合だけを考え、残りを対称性から補うことができる。

・自然に候補となる操作
・変数の入れ替えに対する不変性を確認する
・WLOGが正当化できるか確認する
・基本対称式へ変換する
・平均、和、積などの共通量を用いる
・対称軸、対称中心を設定する
・座標の原点や軸を対称性に合わせる
・均等配置や等号成立条件を疑う
・対称な場合を固定して自由度を減らす
・候補の優先順位
・本当に対称性があるかを確認する
・変数や対象の交換が許されるかを確認する
・共通量へまとめられるかを確認する
・WLOGで場合を減らせるか確認する
・均等配置や平均化が有効か検討する

非発動条件

次の場合は、表面的に似ていても対称性を仮定してはならない。

・一部の変数だけ定義域が異なる
・一部の点や辺に特別な条件が付いている
・目的関数が特定の変数だけを特別扱いしている
・変数を入れ替えると条件が変化する
・図形が見た目だけ対称で、与えられた長さや角度が非対称である

よくある誤用

・対称だから最大値や最小値が必ず等分点で生じると決めつける
・対称式であることだけを理由に a=b=c と置く
・WLOGの根拠を示さず、一つの場合だけを扱う
・図形の見た目だけで対称性を仮定する

検証時の問い

・何を入れ替えても条件が変わらないのか
・求める量も同じ変換で不変なのか
・対称性によって何個の自由度を減らせるのか
・対称性を使わない場合より本当に簡潔になるか

復習カード
条件：
対象の入れ替えや変換で条件が不変

解釈：
対象を個別に区別する必要がない可能性がある

着眼点：
対称性

候補操作：
WLOG、基本対称式、平均化、対称な座標設定

OS-002 存在条件
定義

ある対象、値、解、点、配置が少なくとも一つ存在するための条件を扱う着眼点である。

存在条件は、値域、交点、構成可能性、符号変化、整数条件などへ翻訳される。

・発火条件
・実数解をもつ
・解が存在する
・少なくとも一つ存在する
・交点をもつ
・条件を満たす点が存在する
・ある実数、整数、自然数が存在する
・図形を構成できる
・確率が正である
・ある範囲の値をとる
・条件の解釈

「存在する」という文言は、何らかの対象が許される範囲に入ることを意味する。

方程式では解の存在、関数では値域や交点、図形では構成条件、整数では合同条件や因数条件として現れる。

自然に候補となる操作

・判別式
・値域
・グラフの交点
・単調性
・連続性
・中間値の定理
・定義域の確認
・構成例を一つ与える
・整数条件では合同式、約数、評価
・図形では三角不等式、交点条件、距離条件

候補の優先順位

・何の存在が問われているかを特定する
・定義域や必要条件を確認する
・二次式なら判別式を検討する
・関数なら値域または交点を検討する
・連続性が使えるなら符号変化を確認する
・離散的な対象なら構成または有限化を検討する

非発動条件

・単なる恒等式
・既に存在が保証されており、個数だけが問われる問題
・存在ではなくすべての場合の成立が問われる問題
・必要条件だけを求める問題
・よくある誤用
・判別式を使える形でないのに、存在条件という理由だけで判別式を使う
・必要条件を得ただけで存在が示されたと判断する
・定義域を確認せずに解の存在を論じる
・交点の候補を求めただけで、実際に条件を満たすと判断する

検証時の問い

・存在を示した対象は定義域に入っているか
・得られた条件は必要条件か、十分条件か、必要十分条件か
・実際に一つ構成できるか
・境界値で存在するか
・重解や接点を見落としていないか

復習カード
条件：
実数解、交点、点、整数などが存在する

解釈：
存在可能な範囲または構成条件を求める

着眼点：
存在条件

候補操作：
判別式、値域、交点、連続性、構成、合同式

OS-003 一意性
定義

条件を満たす対象がただ一つに決まることを扱う着眼点である。

存在条件に加えて、複数の候補が生じないことを証明する必要がある。

発火条件

・解がただ一つ
・一意に定まる
・唯一存在する
・交点が一つだけ
・点、直線、円、接線などがただ一つ定まる
・方程式の実数解が一個
・最大値または最小値をとる点が一つ

条件の解釈

一意性は、存在と排他性の二つから成る。

少なくとも一つ存在する
かつ
二つ以上は存在しない

したがって、存在証明だけでは不十分である。

自然に候補となる操作

・単調性
・狭義単調性
・判別式が0
・接する条件
・差を取って0になる条件を調べる
・仮に二つ存在するとして矛盾を導く
・凸性または狭義凸性
・一対一対応
・逆関数
・固定点の一意性

候補の優先順位

・存在が保証されているか確認する
・単調性で二つ以上存在しないことを示せるか確認する
・二次式なら重解条件を検討する
・図形なら接する条件や決定条件を確認する
・二つ存在すると仮定して差や矛盾を調べる
・非発火条件
・単に解を一つ求める問題
・複数解のうち一つを示せばよい問題
・最大値そのものだけを求め、達成点の一意性が不要な問題
・よくある誤用
・解を一つ見つけただけで一意と判断する
・判別式が0でない一般の方程式に重解の議論を適用する
・単調増加ではなく非減少であるだけなのに一意性を結論する
・存在を証明せず、一意性だけを論じる

検証時の問い

・存在と一意性を分けて証明したか
・単調性は狭義である必要がないか
・境界上に別の解が存在しないか
・場合分けによって複数の候補が残っていないか

復習カード
条件：
ただ一つ、一意、唯一

解釈：
存在と排他性の両方が必要

着眼点：
一意性

候補操作：
狭義単調性、重解、接点、背理法、一対一性

OS-004 恒成立・全称条件
定義

定義域内のすべての対象について条件が成立することを扱う着眼点である。

全称条件では、最も成立しにくい場合、すなわち最悪条件を調べる必要がある。

発動条件

・すべての実数 x に対して
・任意の x に対して
・常に成り立つ
・どのように選んでも成り立つ
・すべての自然数 n について
・任意の点、直線、三角形について
・恒等的に成立する

条件の解釈

全称条件は、一つの例を確認するだけでは不十分である。

対象全体で成立するための最小値、最大値、符号、係数、構造を調べる必要がある。

自然に候補となる操作

・差を取って符号を調べる
・最大値または最小値を求める
・二次式の符号と判別式
・グラフの上下関係
・恒等式なら係数比較
・数学的帰納法
・一般の対象を文字で置く
・最悪条件を特定する
・境界および等号成立条件を確認する
・候補の優先順位
・定義域を確認する
・「恒等式」か「恒不等式」かを区別する
・差を取って符号問題へ変換する
・最大最小または判別式を検討する
・自然数全体なら帰納法や差分を検討する
・境界値を確認する

非発動条件

・ある一つの値についてのみ成立する問題
・存在条件
・有限個の候補から一つ選ぶ問題
・単なる数値計算
・よくある誤用
・いくつかの具体例だけで全称命題を証明したと考える
・係数比較を恒等式でない問題に使う
・定義域を広げて議論する
・判別式条件の等号を落とす
・最小値または最大値を確認せずに符号を判断する

検証時の問い

・定義域のすべてを覆っているか
・境界値を含んでいるか
・恒等式と方程式を混同していないか
・一方向の含意だけで終わっていないか
・等号成立の場合を確認したか

復習カード
条件：
任意、すべて、常に

解釈：
対象全体で成立するための最悪条件を調べる

着眼点：
恒成立・全称条件

候補操作：
差の符号、最大最小、判別式、係数比較、帰納法

OS-005 境界・極値
定義

許される範囲の端、条件が切り替わる点、または量が最大・最小となる位置に注目する着眼点である。

境界は、定義域の端点だけでなく、場合分けの境目、等号成立点、接点、退化する配置も含む。

発動条件

・最大値、最小値
・値域、範囲
・上限、下限
・等号成立条件
・区間が指定されている
・パラメータによって解の個数が変化する
・図形が退化する可能性がある
・条件が成立する範囲の端を求める
・場合分けの切替点がある

条件の解釈

極値は、内部の停留点だけでなく、境界でも生じる。

したがって、微分して導関数が0となる点だけを調べるのでは不十分である。

・自然に候補となる操作
・定義域の端点確認
・微分と増減表
・平方完成
・不等式評価
・相加相乗平均
・等号成立条件
・判別式が0となる境界
・接する条件
・場合分けの切替点
・図形の退化配置
・コンパクトな範囲での連続性
・候補の優先順位
・定義域と境界を明示する
・端点を確認する
・内部の停留点を調べる
・場合分けの境目を調べる
・等号成立条件を確認する
・値が実際に達成されるか確認する

非発動条件

・範囲が開いており、最大値や最小値が存在しない場合
・単なる方程式の解法
・局所的な値だけを求める問題
・境界を持たない対象で、他の構造が本質の場合
・よくある誤用
・微分して導関数が0となる点だけを調べる
・端点を確認しない
・上限と最大値、下限と最小値を混同する
・不等式の等号成立条件を確認しない
・定義域外の点を極値候補に含める
・場合分けの境目で値が不連続になる可能性を無視する

検証時の問い

・定義域の端点をすべて調べたか
・極値候補を漏れなく列挙したか
・最大値または最小値は実際に達成されるか
・等号成立条件は元の条件を満たすか
・場合分けの境界で式の値が一致するか

復習カード
条件：
最大、最小、範囲、等号、切替点

解釈：
内部だけでなく境界も極値候補になる

着眼点：
境界・極値

候補操作：
端点、微分、平方完成、不等式、接点、退化配置
